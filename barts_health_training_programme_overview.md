## Module 1 — Data Access, Cleaning & Analysis

**Duration:** 2.5 hours (5 × 30-minute blocks)
**Platform:** Excel Online

### Learning Goals

By the end of this module, participants will be able to:

- Access and import data from multiple sources into Excel Online
- Identify and resolve common data quality issues (duplicates, blanks, inconsistent formatting)
- Use core formulas to interrogate and summarise data
- Understand the difference between VLOOKUP and XLOOKUP, and know when to use each
- Structure cleaned data in a way that is ready for analysis and reporting

### Main Areas Covered

- Navigating Excel Online and connecting to data sources (SharePoint, OneDrive, exported CSVs)
- Data cleaning techniques: removing duplicates, fixing formatting, handling blank cells
- Core formulas: SUM, AVERAGE, COUNTIF, SUMIF, IF, IFERROR
- VLOOKUP vs XLOOKUP — understanding the key differences and limitations of each
- Structured tables and named ranges for cleaner, more reliable formulas
- Filters, sorting, and simple PivotTables for summarising data

### 30-Minute Block Breakdown

**Block 1 — Getting Data into Excel Online (09:00–09:30)**
- How Barts Health data flows: SharePoint, OneDrive, exported system reports
- Opening and importing files in Excel Online
- Understanding how data arrives "in the wild" — messy, inconsistent, incomplete
- Hands-on: open a provided anonymised Barts Health dataset and identify problems

**Block 2 — Cleaning & Structuring Data (09:30–10:00)**
- Removing duplicates and handling blank rows or cells
- Fixing number/date/text formatting inconsistencies
- Converting a data range to a structured Excel Table
- Hands-on: clean the dataset from Block 1, ready for analysis

**Block 3 — Core Formulas for Analysis (10:00–10:30)**
- SUM, AVERAGE, COUNT, COUNTIF, SUMIF with real dataset examples
- IF and nested IF for basic logic
- IFERROR to handle formula errors gracefully
- Hands-on: build a summary section using formulas on the cleaned dataset

**Block 4 — VLOOKUP vs XLOOKUP (10:30–11:00)**
- What a lookup formula does and why it matters
- VLOOKUP: syntax, limitations (left-to-right only, fragile column indexing)
- XLOOKUP: syntax, flexibility (any direction, not-found handling, exact/approximate match)
- Side-by-side comparison using the same data scenario
- Hands-on: replace a VLOOKUP with XLOOKUP and observe the difference

**Block 5 — PivotTables & Summary Structures (11:00–11:30)**
- Building a simple PivotTable from a structured table
- Grouping, filtering, and refreshing pivot data
- When to use a PivotTable vs a formula-based summary
- Hands-on: create a PivotTable summarising the cleaned dataset by category

### Conclusion

By the end of Module 1, participants have a working, clean dataset summarised by key categories — ready to be visualised in Module 2. The key habit instilled is: **clean first, analyse second**. Participants understand that reliable insight starts with reliable data, and that XLOOKUP is the modern, more robust replacement for VLOOKUP in Excel Online.

## Module 2 — Data Visualisation & Storytelling

**Duration:** 1.5 hours (3 × 30-minute blocks)
**Platform:** Excel Online, Power BI (overview), Microsoft Fabric (overview)

### Learning Goals

By the end of this module, participants will be able to:

- Select the right chart type for different types of data and messages
- Remove clutter from charts and add context that aids understanding
- Structure a data narrative using the Context → Insight → Recommendation framework
- Write chart titles and slide headlines that communicate the finding, not just the topic
- Understand how Power BI and Microsoft Fabric outputs can complement Excel charts in presentations

### Main Areas Covered

- Principles of effective data visualisation
- Choosing the right chart: bar, line, pie, scatter — and when each is appropriate (and when not)
- Decluttering charts: removing gridlines, legends, and noise that distract
- The Context → Insight → Recommendation narrative framework
- Writing actionable headlines vs descriptive labels
- Introduction to Power BI and Fabric as reporting layers used across Barts Health

### 30-Minute Block Breakdown

**Block 1 — Choosing the Right Chart (11:45–12:15)**
- Bar charts for comparison, line charts for trends, pie charts for composition (and their limits)
- Scatter charts for relationships between two variables
- Common mistakes: using pie charts for too many categories, dual-axis charts that mislead
- Hands-on: given four data scenarios, participants select and justify a chart type for each

**Block 2 — Building & Decluttering Charts in Excel Online (12:15–12:45)**
- Inserting charts from a PivotTable or data range
- Formatting for clarity: removing chart borders, softening gridlines, choosing accessible colours
- Writing chart titles that tell the story ("Attendance declined 12% in Q3" not "Attendance by Quarter")
- Hands-on: build and format two charts from the Module 1 dataset

**Block 3 — Telling the Data Story (12:45–13:15)**
- The Context → Insight → Recommendation framework: what it is and why it works for senior audiences
- Translating a chart into a slide: one message per slide, lead with the insight
- Introduction to Power BI and Microsoft Fabric: how dashboards from these tools are used at Barts Health and how outputs can be embedded or referenced in presentations
- Hands-on: take one chart and build a single-slide summary using the narrative framework

### Conclusion

By the end of Module 2, participants can turn a cleaned dataset into a clear, insight-led visual and frame it as a coherent message for a senior stakeholder. The key principle: **a chart without a narrative is just a picture**. The Context → Insight → Recommendation framework gives participants a repeatable structure they can apply to any data communication task.

## Module 3 — Accelerating with Copilot

**Duration:** 1.5 hours (3 × 30-minute blocks)
**Platform:** Microsoft Copilot within Excel Online, PowerPoint Online, Word Online, Outlook, Teams

### Learning Goals

By the end of this module, participants will be able to:

- Write clear, effective prompts for common administrative and data tasks
- Use Copilot in Excel to generate formulas, identify anomalies, and summarise data
- Use Copilot in PowerPoint to create first-draft slides from a dataset or document
- Validate, edit, and take ownership of AI-generated outputs before sharing
- Identify the tasks where Copilot adds the most value versus where manual effort is better

### Main Areas Covered

- How Microsoft Copilot works within the M365 suite and what it can and cannot do
- Effective prompting: specificity, context, constraints, and worked examples
- Copilot in Excel: formula generation, data summarisation, anomaly detection
- Copilot in PowerPoint: slide generation from prompts, documents, and data
- Copilot in Word, Outlook, and Teams: drafting, rewriting, summarising, email and meeting support
- Validating AI outputs: checking for errors, hallucinations, and misrepresentation of data

### 30-Minute Block Breakdown

**Block 1 — Effective Prompting & Copilot in Excel (14:00–14:30)**
- What makes a good prompt: role, context, task, constraints, desired output format
- Common prompting mistakes and how to fix them
- Copilot in Excel: asking Copilot to explain a dataset, suggest a formula, or summarise a column
- Hands-on: use Copilot to generate and explain two formulas from a provided dataset, then validate the output manually

**Block 2 — Copilot in PowerPoint & Word (14:30–15:00)**
- Generating a first-draft slide deck from a prompt or existing document using Copilot
- Refining and editing Copilot-generated slides to meet Barts Health standards
- Copilot in Word: drafting a narrative summary from a data table; rewriting for tone; summarising a long document
- Hands-on: use Copilot to create a first-draft summary slide from the Module 2 dataset, then improve it

**Block 3 — Copilot Across Teams, Outlook & Validation (15:00–15:30)**
- Copilot in Teams: summarising meeting notes and generating action points
- Copilot in Outlook: drafting and refining emails to clinical and operational stakeholders
- Validating AI-generated outputs: what to check before sharing anything Copilot has produced
- Hands-on: draft a short stakeholder email using Copilot, then edit it to reflect the correct tone and data from earlier modules

### Conclusion

By the end of Module 3, participants can use Copilot to meaningfully reduce time spent on routine drafting, formula writing, and summarisation tasks. The key discipline: **Copilot is a first draft, not a final product**. Every output needs a human review before it is shared. Participants leave with a mental map of where Copilot saves time and where they still need to apply their own judgement.

## Module 4 — End-to-End Workflow

**Duration:** 1 hour (2 × 30-minute blocks)
**Platform:** Excel Online, PowerPoint Online, Microsoft Copilot

### Learning Goals

By the end of this module, participants will be able to:

- Apply skills from Modules 1–3 in a single, realistic workflow from start to finish
- Make judgement calls about when to use manual techniques versus Copilot assistance
- Produce a clear, insight-led output to a defined brief under time pressure
- Reflect on their own performance and identify areas to develop further
- Commit to at least one concrete change in how they approach data tasks in their role

### Main Areas Covered

- A timed, realistic Barts Health scenario brief (e.g. monthly performance summary for a clinical director)
- Applying data cleaning, analysis, and XLOOKUP under time pressure
- Selecting and building appropriate visualisations
- Structuring a concise insight-led presentation using the narrative framework
- Using Copilot to accelerate drafting and formatting steps where appropriate
- Peer review and group debrief

### 30-Minute Block Breakdown

**Block 1 — End-to-End Scenario (15:45–16:15)**
- Participants receive a scenario brief and a raw, anonymised Barts Health dataset
- Task: clean the data, analyse it, build relevant visuals, and produce a short insight-led presentation slide or summary — using any tools and approaches from Modules 1–3
- Copilot use is permitted and encouraged where it genuinely saves time
- Facilitator circulates to observe decisions and support where needed

**Block 2 — Peer Review, Debrief & Action Planning (16:15–16:45)**
- Participants share outputs in pairs and give structured feedback: what is clear, what is missing, what would a senior stakeholder question?
- Group debrief: what approaches worked well, where did people get stuck, what would they do differently?
- Personal action planning: each participant identifies one or two specific ways they will apply today's skills in their own role within the next 30 days
- Facilitator summarises key learning and signposts post-session resources

### Conclusion

Module 4 closes the loop on the day's learning. It is deliberately shorter and more open-ended than earlier modules, giving participants the space to apply, struggle, and reflect. The goal is not a perfect output — it is **confident, independent decision-making** across the full data-to-insight workflow. Participants leave with a concrete action plan, not just a set of skills they used once in a training room.

## Post-Session Reinforcement (30 Days)

Following the training day, participants receive structured support to embed skills in their day-to-day roles:

- **Recap guides** for each module — concise one-page references for formulas, chart types, prompting tips, and the narrative framework
- **Challenge tasks** — short, practical exercises based on real Barts Health admin scenarios, one per week for four weeks
- **Copilot prompt library** — a curated set of ready-to-use prompts for common tasks (data summarisation, email drafting, meeting notes, slide generation)
- **30-day check-in** — a brief facilitator-led conversation to measure confidence, surface questions, and identify any further support needed
